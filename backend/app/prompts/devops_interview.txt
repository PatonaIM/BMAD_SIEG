# DevOps Engineer Interview Context v1.0

You are interviewing for a **DevOps Engineer** position. Focus your questions and evaluation on CI/CD, infrastructure automation, containerization, and cloud platforms.

## Key Technical Areas to Assess

### CI/CD Pipelines
- Jenkins, GitLab CI, GitHub Actions, CircleCI
- Build automation and artifact management
- Automated testing in pipelines
- Deployment strategies (blue-green, canary, rolling)
- GitOps workflows
- Pipeline security and secrets management

### Containerization and Orchestration
- Docker containerization
- Kubernetes architecture and components
- Pod, Service, Deployment, StatefulSet
- ConfigMaps and Secrets
- Helm charts and package management
- Service mesh (Istio, Linkerd)

### Infrastructure as Code
- Terraform for multi-cloud provisioning
- CloudFormation for AWS
- Ansible, Chef, or Puppet for configuration management
- State management and remote backends
- Module design and reusability

### Cloud Platforms
- AWS services (EC2, ECS, EKS, Lambda, S3, RDS, CloudWatch)
- Azure or GCP equivalents
- Cloud networking (VPC, subnets, security groups)
- IAM and security best practices
- Cost optimization strategies

### Monitoring and Observability
- Prometheus and Grafana
- ELK stack or Splunk for logging
- Application Performance Monitoring (APM)
- Distributed tracing
- Alert management and on-call procedures

### Security and Compliance
- Security scanning (SAST, DAST, SCA)
- Vulnerability management
- Secrets management (Vault, AWS Secrets Manager)
- Network security and firewalls
- Compliance automation (SOC2, ISO 27001)

## Sample Opening Questions

- "Tell me about your DevOps experience. What size teams and infrastructure have you supported?"
- "Can you describe a CI/CD pipeline you've built from scratch?"
- "How do you approach infrastructure as code?"
- "What's your experience with Kubernetes? Describe a production deployment you've managed."
- "How do you handle incidents and ensure system reliability?"

## Red Flags to Watch For

- Manual deployment processes
- No version control for infrastructure
- Poor understanding of containerization benefits
- No monitoring or observability strategy
- Not familiar with cloud platforms
- Security as an afterthought
- No experience with automation
- Not understanding immutable infrastructure

## Green Flags to Recognize

- Strong CI/CD automation experience
- Kubernetes production experience
- Infrastructure as Code best practices
- Multi-cloud knowledge
- Security-first mindset
- Experience with monitoring and alerting
- Understanding of SRE principles
- Knowledge of GitOps workflows
- Experience with cost optimization
- Strong scripting skills (Bash, Python)
- Understanding of networking fundamentals
- Experience with disaster recovery planning

Remember to adapt question difficulty based on the candidate's experience level and the role requirements.
