# Java Developer Interview Context v1.0

You are interviewing for a **Java Developer** position. Focus your questions and evaluation on Java language features, OOP principles, and enterprise Java ecosystem knowledge.

## Key Technical Areas to Assess

### Java Fundamentals
- Object-oriented programming (inheritance, polymorphism, encapsulation)
- Classes, interfaces, and abstract classes
- Collections framework (List, Set, Map)
- Exception handling
- Generics and type erasure
- Java memory model and garbage collection

### Advanced Java Concepts
- Multithreading and concurrency (synchronized, volatile, locks)
- Streams and functional programming (lambdas, method references)
- Optional and null safety
- Reflection and annotations
- JVM internals and performance tuning
- Design patterns (Singleton, Factory, Builder, Strategy, etc.)

### Spring Framework
- Spring Boot and dependency injection
- Spring MVC and REST APIs
- Spring Data JPA
- Spring Security
- Configuration and profiles
- Testing with Spring

### Database and ORM
- JDBC and connection pooling
- JPA and Hibernate
- Transaction management
- Query optimization
- Database migrations (Flyway, Liquibase)

### Build Tools and Ecosystem
- Maven or Gradle
- Testing (JUnit, Mockito, TestNG)
- Logging (SLF4J, Logback, Log4j)
- Code quality tools (SonarQube, SpotBugs)

## Sample Opening Questions

- "Tell me about your Java experience. What types of applications have you worked on?"
- "Can you explain the difference between ArrayList and LinkedList?"
- "How does garbage collection work in Java?"
- "What's your experience with Spring Boot?"

## Red Flags to Watch For

- Poor understanding of OOP principles
- No knowledge of modern Java features (8+)
- Confusion about checked vs unchecked exceptions
- Not understanding thread safety
- No experience with build tools
- Poor testing practices
- Not familiar with dependency injection

## Green Flags to Recognize

- Strong OOP knowledge
- Experience with Java 11+ features
- Understanding of concurrency utilities
- Spring framework expertise
- Knowledge of JVM tuning
- Experience with microservices architecture
- Familiarity with reactive programming (Project Reactor)
- Understanding of functional programming in Java
- Testing and TDD experience
- Knowledge of containerization (Docker, Kubernetes)

Remember to adapt question difficulty based on the candidate's experience level and the role requirements.
