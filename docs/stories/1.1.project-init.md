### Coding Standards

**Backend Standards** [Source: architecture/backend/12-coding-standards.md]
- **Type hints required** on all functions: `async def func(param: Type) -> ReturnType:`
- **Naming:** snake_case for functions/variables, PascalCase for classes, UPPER_SNAKE_CASE for constants
- **Async everywhere:** All I/O operations must use async/await
- **Structured logging:** Use `logger.info("event_name", key=value)` not print()
- **Error handling:** Wrap external API calls in try-except with appropriate HTTPException
- **Import organization:** Standard library → Third-party → Local (separated by blank lines)

**Frontend Standards** [Source: architecture/frontend/11-frontend-developer-standards.md]
- **TypeScript strict mode:** No `any` types without justification
- **Component naming:** PascalCase, files match component names
- **Imports:** Use `@/` alias for absolute imports
- **Props:** Destructure in function signature, prefix booleans with `is/has/should`
- **State:** Zustand for shared state, useState for local UI state
- **Effects:** Clean up side effects, include all dependencies
- **Accessibility:** Include proper ARIA labels, ensure keyboard navigation

### Testing

**Backend Testing Standards** [Source: architecture/backend/13-test-strategy.md]
- **Framework:** pytest 7.4+ with pytest-asyncio 0.21+
- **Test Structure:** 60% unit, 30% integration, 10% e2e
- **File Location:** `tests/unit/`, `tests/integration/`, `tests/e2e/`
- **Naming:** `test_{module}_test.py`, test functions prefixed with `test_`
- **Fixtures:** Defined in `tests/conftest.py`
- **Async Tests:** Decorated with `@pytest.mark.asyncio`
- **Mocking:** Use `pytest-mock` for test doubles
- **Coverage:** Target 80% minimum coverage

**Frontend Testing Standards** [Source: architecture/frontend/09-testing-requirements.md]
- **Framework:** Vitest 1.0+ (Jest-compatible API, native Vite integration)
- **Component Testing:** React Testing Library 14.1+ (user-centric testing)
- **Setup:** `tests/setup.ts` with test environment configuration
- **File Location:** Colocate tests with components (`ComponentName.test.tsx`)
- **Test Structure:** Arrange-Act-Assert pattern
- **Providers:** Wrap components with ThemeProvider for MUI components
- **Mocking:** Mock external dependencies (API calls, routing, global state)
- **Coverage Goal:** 80% code coverage
- **Run Commands:**
  - `npm run test` - Run tests in watch mode
  - `npm run test:ui` - Run tests with UI
  - `npm run test:coverage` - Run with coverage report
